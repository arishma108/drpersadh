JPMorgan Chase & Co. — Engineering Apprenticeship Log
Month 6 — The Final Project
System observability dashboard
Built a dashboard visualising system metrics from logs: response times, error counts, transaction volumes. Simple in concept, powerful in impact. A senior engineer reminded me: good engineering is not brilliance, it is clarity for everyone else.
Month 5 — Logs, Logs, and More Logs
Performance investigation cycle
Traced a production slowdown that only appeared under specific dataset thresholds. CPU and queries looked fine until the edge case revealed itself. We fixed it and added monitoring. This is forensic engineering, not coding.
Month 4 — Inside the Machine
Enterprise system thinking
Sprint meetings exposed the real world of engineering: risk, compliance, system resilience. Small code changes trigger architectural conversations. Production systems behave like living organisms, not textbooks.
Month 3 — First Real Bug Fix
Production incident resolution
Fixed incorrect reporting caused by null handling in SQL. No celebration, just correctness restored. But there is weight in knowing your code touched financial truth.
Month 2 — Corporate Hello World
Workflow immersion
Learned the real workflow: pull, reproduce, fix, commit, review. Code reviews were less about approval and more about reasoning. Reliability beats cleverness every time.
Month 1 — The Quiet Shock of Scale
System onboarding
Enterprise systems revealed themselves immediately: layered architectures, strict access control, decades of evolution. I spent most of the time observing senior engineers inside production systems before writing anything.
